Title: v0.0.3-alpha
Tag: v0.0.3-alpha
Status: alpha hardening release

## Summary

v0.0.3-alpha is an alpha hardening release built on v0.0.2-alpha. It adds API drift gates, JSONL storage guardrails, deterministic fault-injection gates, TypeScript obligation registry and in-toto/DSSE shape-helper parity, release provenance artifact gates, and storage compatibility policy.

This remains an alpha substrate release. It does not change the public release boundary: Attestplane is not production-ready, not compliance-ready, and not a certification system. CLI `attestplane verify` remains `chain_report_only`.

## Highlights

- P2.1 public API drift gate for Python and TypeScript public surfaces.
- P2.2 alpha JSONL storage guardrails for newline-terminated append, corruption reporting, inspect/export behavior, and single-writer semantics.
- P2.3 deterministic fault-injection gate covering fail-closed invariants across canonicalization, hash chains, payload validators, proof/verifier predicates, settlement, anchoring, JSONL storage, and CLI claim boundaries.
- P2.4 TypeScript obligation registry plus in-toto/DSSE shape-helper parity, with explicit scope limits.
- P2.5 release provenance artifact gate for manifest, checksum helper, optional keyless signing dry-run, and no-go supply-chain claims.
- P2.6 storage compatibility and migration policy for JSONL records, scan issues, export behavior, and future storage backends.

## CLI Verify Scope

CLI `attestplane verify` remains `chain_report_only`. It must not be treated as a full ProofBundle verifier, signed verifier, anchored verifier, compliance verifier, or certification verifier.

## Migration Notes

- No public API break is intended.
- Public API manifests still use v0.0.2-alpha as the baseline release for drift detection.
- JSONL rows without `storage_record_version` remain readable as the current implicit v1 format.
- Unsupported explicit `storage_record_version` values fail closed.
- The v0.0.3-alpha release artifact manifest is included in the tag.
