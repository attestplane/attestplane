# v0.8.0-rc.1 Draft Release Notes

`v0.8.0-rc.1` is the first Attestplane release-candidate cut planned after
`v0.8.0-beta.0`.

## Status

Release candidate. API and wire-format surfaces are frozen pending GA, subject
to explicit `rc.N` bumps for any breaking correction. This is not GA, not a
production-readiness commitment, not a compliance certification, and not legal
advice.

## Changes Since v0.8.0-beta.0

- Adds the release-candidate compatibility policy for wire-format and SDK
  surface freeze.
- Adds ADR-0016 documenting the RC API-freeze decision.
- Expands verifier conformance with tampered-report, schema-version-skew, and
  malformed policy-trace reference cases.
- Updates npm release workflow policy so RC packages publish under the `rc`
  dist-tag and prerelease packages cannot be accidentally published with
  `latest`.
- Updates security support language for beta, RC, and future GA lines.

## Registry Plan

- PyPI: `attestplane==0.8.0rc1`.
- npm: `@attestplane/attestplane@0.8.0-rc.1` under the `rc` dist-tag.
- npm `beta` remains available for `0.8.0-beta.0`.
- npm `alpha` remains frozen at the last alpha line until retired or
  documented otherwise.
- npm `latest` movement is a maintainer policy decision and must be recorded
  separately from package publication.

## Required Validation

- Local Python verifier conformance.
- Local TypeScript verifier conformance.
- Fixture hash lock validation.
- Python and TypeScript SDK gates.
- GitHub Actions: `ci`, `sdk-python`, `sdk-typescript`,
  `cross-sdk-roundtrip`, `verifier-conformance`, `invariants`, `codeql`,
  `osv-scanner`, `sbom`, `reproducible-build`, and `ossf-scorecard`.
- Registry-installed verifier conformance and cross-SDK round-trip after
  publication.

## Rollback

Published RC artifacts are immutable. If an RC is defective:

- PyPI artifact may be yanked, not deleted.
- npm artifact should be deprecated and dist-tags rewound.
- A fixed release must use `v0.8.0-rc.2` or later.
