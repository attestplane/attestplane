# Attestplane v0.8.5 Controlled Availability

Date: 2026-05-20

`v0.8.5` is the Controlled Availability package cut promoted from the
`v0.8.5-rc.5` evidence gate. It is a suffix-free stable package version so
default PyPI and npm installs can resolve to the same release:

```bash
pip install attestplane
npm install @attestplane/attestplane
```

This is not GA, not a production-readiness claim, not a regulatory
certification, and not a legal-compliance claim. It remains a pre-GA substrate
release for controlled consumers who need the frozen `AP-EVD/1.0` wire format,
chain/report-oriented verifier predicates, and cross-SDK conformance surface.

## Registry Channels

- PyPI: `attestplane==0.8.5`
- npm: `@attestplane/attestplane@0.8.5`
- npm `latest`: `0.8.5`
- npm `ca`: `0.8.5`
- npm `rc`: remains pinned to `0.8.5-rc.5`
- npm `beta`: remains pinned to `0.8.0-beta.0`

## Changes Since v0.8.0-beta.0

- Public SDK export and wire-format freeze under ADR-0016.
- GitHub Actions `release-cd` as the package publication entrypoint.
- RC-aware npm dist-tag validation and prerelease/latest guardrails.
- Verifier conformance expansion for tampered reports, version skew,
  malformed policy-trace references, and missing-event cases.
- Stable verifier error taxonomy and retention/deletion proof marker checks.
- GA/CA cut criteria and evidence packet for the `v0.8.5-rc.5` gate.

## Verification Evidence

The pre-cut evidence is recorded in
[`docs/validation/ga_ca_cut_evidence_20260520.md`](../validation/ga_ca_cut_evidence_20260520.md).
Post-cut verification must confirm:

- clean PyPI default install resolves to `0.8.5`;
- clean npm default install resolves to `0.8.5`;
- verifier conformance passes against the package surface;
- Python/TypeScript cross-SDK roundtrip stays byte-identical.

## Rollback Notes

Do not delete or reuse published package versions. If default npm installs need
to roll back, move `latest` and `ca` through the documented npm-management path
and record the observed registry state. If PyPI must be withdrawn, use PyPI
yank rather than deleting artifacts.
