# v1.6.2

`v1.6.2` is an automated suffix-free stable package cut from autodev-train.

## User-Visible Delta

The single user-visible change in `v1.6.2` is release-planning automation
reliability: planned-task issues created from Opus consultations are now
re-fetched from GitHub before downstream runner consumption. That closes the
first-run race where the runner could create planned-task issues successfully
but then hand an empty issue list to the next stage, making downstream
automation report zero newly created issues until a later poll.

Planning context: [Issue #113](https://github.com/attestplane/attestplaneissues113);
boundary-audit pattern:
[Issue #108](https://github.com/attestplane/attestplaneissues108);
related local planned tasks:
[Issue #117](https://github.com/attestplane/attestplaneissues117) and
[Issue #118](https://github.com/attestplane/attestplaneissues118).

## Infrastructure

- CI-only release-train hygiene made the stable-train git proxy strategy
  explicit.
- CI-only runner hygiene uses local Python on the Opus runner. These items are
  operational improvements only; they do not change SDK, CLI, verifier, schema,
  package, registry, signing, or compliance behavior.

## Highlights

- Publishes suffix-free Python and npm package versions through GitHub
  `release-cd`.
- Advances PyPI default installs because PyPI has no dist-tag concept.
- Advances npm `latest` through the stable release workflow.
- Does not move npm `ca`; CA remains a separate manual dist-tag decision.
