# v0.0.5-alpha

`v0.0.5-alpha` is a verifier-conformance and runtime-evidence tightening
release after the frozen `v0.0.4-alpha` line.

## Highlights

- Added stable `VERIFY_*` verifier error taxonomy for deterministic JSON output.
- Added Python and TypeScript retention/deletion proof helpers under the
  ADR-0015 commit-then-redact profile.
- Added optional `retention_proofs` in ProofBundle v1, with verifier checks for
  shape and event-hash references.
- Added cross-language verifier conformance vectors and CI workflow.
- Kept the TSA provider interface as a pluggable source abstraction without
  changing the verifier trust root.
- Added a threat-model snapshot and non-goal boundaries for this alpha line.

## Explicit Boundaries

This release does not claim:

- EU AI Act compliance,
- GDPR compliance,
- legal certification,
- production readiness,
- certified provenance,
- SLSA L3,
- production-grade supply-chain security, or
- long-term archival trust guarantees.

The verifier is still an offline, deterministic engineering verifier. Hosted
services are not required as the trust root.

## Release-Integrity Limitation

The v0.0.5-alpha local release gate generated checksums and CycloneDX SBOMs.
Local Sigstore/cosign keyless signing was attempted but required an interactive
device-flow session and did not complete in the non-interactive release run.
Therefore this release should not be described as signed, certified provenance,
or SLSA L3.
