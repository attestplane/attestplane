## User-Visible Delta

This support note extends the open docs work in [#211](https://github.com/attestplane/attestplane/issues/211),
[#157](https://github.com/attestplane/attestplane/issues/157), and
[#246](https://github.com/attestplane/attestplane/issues/246). It is docs-only
support material and stays gated behind the underlying product issues landing.

- ISSUE 1, positive forward-compat path: [`docs/schema/verify-json.md`](../schema/verify-json.md)
  records that additive unknown bundle fields are accepted when the rest of
  the bundle is valid. That is the user-visible forward-compat behavior for
  additive-only changes.
- ISSUE 2, `taxonomy_version` surfacing: [`docs/cli/verify-json.md`](../cli/verify-json.md),
  [`docs/schema/verify-json.md`](../schema/verify-json.md), and
  [`docs/errors.md`](../errors.md) all surface the same `taxonomy_version = 1`
  verifier taxonomy so `verify --json`, `verify --explain`, and the canonical
  reason-code table stay aligned.
- ISSUE 3, `--json` exit-code contract: [`docs/cli/verify-json.md`](../cli/verify-json.md)
  documents `exit_code` as the process gate and tells consumers to branch on
  shell status first, then inspect `status`, `ok`, `checks[]`, and the
  detailed signature/anchor status fields.
- `verify-proofbundle` now uses a deterministic four-way exit-code map for CI
  gating: `0` valid, `2` invalid signature/anchor, `3` schema/output-contract
  error, and `4` quarantined/anchor-unverifiable. That is a user-visible
  behavior change from the older `0/1/2` mapping and should be called out in
  the v1.8.x delta notes.
- Out of scope for this support task: `#63` remains open and intentionally
  unscheduled for `v1.8.4` because release-signing is a
  release-integrity/governance track, not a daily product increment.

## Highlights

- Publishes suffix-free Python and npm package versions through GitHub `release-cd`.
- Advances PyPI default installs because PyPI has no dist-tag concept.
- Advances npm `latest` through the stable release workflow.
- Does not move npm `ca`; CA remains a separate manual dist-tag decision.
