# v1.7.x User-Visible Delta

This support-task delta extends Issue #157. It records the verifier, CLI,
and schema-version surface that became visible in the v1.7.x train without
replacing the product work tracked in the source issues.

## Changes Since #157

- Reason-code taxonomy reference: `docs/errors.md` now documents the SDK-public
  `att.verify.*` rejection reason surface introduced by Issue #172.
- `verify --explain`: adds a human-oriented explanation path for rejected
  verifier results so operators can read the failure without parsing the JSON
  payload by hand.
- `verify --json`: emits the machine-readable verifier result, including
  `error_code`, `primary_reason`, `secondary_reasons`, and the derived
  `reasons[]` compatibility view.
- `schema_version` policy: bundle schema versions remain separate from the
  verifier reason-code taxonomy, missing versions now surface as
  `att.verify.schema_version_missing`, and unsupported versions should surface
  as `att.verify.schema_version_unsupported`.
- Forward-compatible additive fields: bundles that only add unknown fields are
  now accepted by the verifier. The verifier preserves the input object
  verbatim and ignores the extra fields during gating.
- Migration note: the legacy human-readable reason strings remain available for
  one minor release as deprecation aids; consumers should branch on
  `primary_reason`, `secondary_reasons`, or `reasons[]`.
- negative conformance vectors: see `tests/conformance/vectors/negative/`
  for the failure cases that pin required-field, signature-missing, and
  schema-version behavior.

## CI Gating Example

Use the exit code as the gate and derive a `reasons[]` list from the JSON
payload:

```sh
attestplane verify "$bundle" --json > verify.json
rc=$?

reasons=$(jq -cr '[.primary_reason] + .secondary_reasons | map(select(. != null))' verify.json)

if [ "$rc" -ne 0 ]; then
  printf '%s\n' "$reasons" | jq -r '.[]'
  exit "$rc"
fi
```

In this pattern, `reasons[]` is the derived list of verifier rejection
reasons. `primary_reason` is the first machine-readable reason, and
`secondary_reasons` preserves the remaining failed checks in order.
