# v1.7.0

`v1.7.0` is an automated suffix-free stable package cut from autodev-train.
It is the first stable release since `v1.5.0` to carry user-visible product
changes. The cadence guard from `5b32c86` blocks productless stable cuts, and
Issue #119 restored product-bearing autodev planning before this release was
prepared.

## Changes Since Previous Stable

- `3f551d9` requires strict consumers to reject empty proof bundles on demand.
- Issue 1 tightens the minimum signed-attestation schema: a strict bundle must
  include a signed attestation with signature payload material and a canonical
  subject digest.
- Issue 3 adds the SDK migration path through
  `attestplane.sdk.ProofBundleBuilder.minimal(subject_digest, signer)` plus
  typed `EmptyProofBundleError` and `IncompleteProofBundleError` handling.
- `5b32c86` blocks productless stable releases, preserving the v1.7.0 delta as
  a product-bearing stable cut.
- Issue #119 prioritizes product autodev plans so the stable train does not
  advance on release-prep-only commits.

## What Integrators Must Do

1. Stop emitting or accepting empty proof bundles in strict verification paths.
2. Emit at least one minimum-valid signed attestation with subject digest
   material.
3. Migrate SDK callers to `ProofBundleBuilder.minimal(...)` and catch
   `EmptyProofBundleError` / `IncompleteProofBundleError`.

Planning context: [Issue #120](https://github.com/attestplane/attestplane/issues/120);
implementation summary task:
[Issue #125](https://github.com/attestplane/attestplane/issues/125).

## Highlights

- Publishes suffix-free Python and npm package versions through GitHub
  `release-cd`.
- Advances PyPI default installs because PyPI has no dist-tag concept.
- Advances npm `latest` through the stable release workflow.
- Does not move npm `ca`; CA remains a separate manual dist-tag decision.
