# v1.7.x Changes

This page is the narrow v1.7.x change/reference surface for user-visible SDK
and CLI deltas. It extends the v1.7.0 summary in
[`docs/release-notes/v1.7.0.md`](../release-notes/v1.7.0.md) and the SDK
reference note in [`docs/contributor/api-reference.md`](../contributor/api-reference.md);
it does not replace those documents.

## v1.7.1 SDK + CLI Delta

The v1.7.1 documentation delta covers three local product increments that build
on the Issue #125 v1.7.0 minimum proof-bundle contract:

- Additive canonicalization conformance vectors under
  `tests/conformance/vectors/canonicalization/` for the strict minimum
  proof-bundle helper and verifier path. The positive vectors cover helper
  output for canonical JSON boundaries, duplicate-key control, signed-int64
  payload boundaries, and NFC payload strings. The negative vectors cover raw
  duplicate JSON keys, BOM/trailing bytes, signed-int64 overflow, and NFD
  payload strings. These vectors are additive; existing v1.7.x fixture hashes
  are not rewritten.
- Two explicit `attestplane verify` opt-in flags:
  `--require-non-empty` rejects proof bundles with zero events, and
  `--strict-schema` enforces the minimum signed-attestation schema. Default
  `attestplane verify <bundle>` behavior remains chain/report-oriented and does
  not become full ProofBundle, signature, anchor, or compliance certification
  verification.
- A signed-schema round-trip regression under
  `tests/verifier/test_signed_schema_roundtrip.py` and
  `tests/conformance/test_signed_schema_conformance_roundtrip.py`. The
  regression locks the locally accepted signed fixture against SDK
  serialization drift; it is test coverage, not a new runtime verification
  claim.

### SDK

Use `ProofBundleBuilder.minimal(...)` when SDK callers need the minimum
signed-attestation bundle shape accepted by the strict v1.7.x verifier
contract.

```python
from attestplane.sdk import (
    EmptyProofBundleError,
    IncompleteProofBundleError,
    ProofBundleBuilder,
    verify_minimum_bundle,
)
from attestplane.signing import InMemoryKeyProvider, Signer

subject_digest = "3f551d9" + "0" * 57
signer = Signer(
    chain_id="example-minimum-bundle",
    key_provider=InMemoryKeyProvider(seed=b"\x12" * 32),
)

try:
    bundle = ProofBundleBuilder.minimal(subject_digest, signer)
    result = verify_minimum_bundle(bundle)
except EmptyProofBundleError:
    raise
except IncompleteProofBundleError:
    raise
else:
    assert result.ok
```

### CLI

Use the opt-in flags when automation wants the v1.7.x minimum proof-bundle
contract instead of the default chain/report-oriented verification mode.

```bash
attestplane verify tests/fixtures/v1.7.0_signed.json --require-non-empty --strict-schema
```

The CLI still reports that `attestplane verify` is chain/report-oriented. The
strict flags do not perform cryptographic signature verification, anchor
verification, network access, or compliance certification.
