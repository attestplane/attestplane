# v1.6.0

`v1.6.0` is an automated suffix-free stable package cut from
autodev-train.

## User-Visible Delta

The local `v1.5.0..v1.6.0` boundary contains 18 non-release-prep commits. They
are release-automation hardening, not SDK or verifier behavior changes.

### Autodev Planning Loop Hardening

- Stable planning now consults Opus before issuing accepted plans, and the
  release tiers share one plan-issuance path instead of diverging by tier.
- Accepted plans can be converted into planned-task issues, with open GitHub
  issues included and planned issues reloaded from GitHub before downstream
  runner consumption.
- Release planning schema and fan-out behavior are unified, and structured
  autodev train events make the planning loop easier to audit.

### Daily Architecture Audit Fan-Out

- Daily architecture audit plans are generated automatically.
- Architecture plans are fanned out for runner execution.
- Architecture planning runs on the Opus runner with local Python selected
  explicitly, so the audit path matches the intended consultation tier.

### Release Plumbing Robustness

- The stable-train git proxy strategy is explicit.
- Idle cadence checks are skipped before remote probing, avoiding misleading
  cadence decisions before the train knows whether the remote is reachable.
- Release-planning scripts are importable in CI.
- Transient Scorecard link failures are ignored as infrastructure noise rather
  than release-train product behavior.
- Markdown and plan-parser cleanup keeps generated planning artifacts lintable.

## Compatibility

No breaking changes. No public API surface changes.

This release does not change SDK APIs, CLI behavior, verifier semantics,
schemas, package names, release asset names, registry behavior, signing policy,
release-gate policy, or claim-safety policy. No application upgrade action is
required.

## Highlights

- Publishes suffix-free Python and npm package versions through GitHub
  `release-cd`.
- Advances PyPI default installs because PyPI has no dist-tag concept.
- Advances npm `latest` through the stable release workflow.
- Does not move npm `ca`; CA remains a separate manual dist-tag decision.
